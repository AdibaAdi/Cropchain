# Comparative Analysis Notes

## 1. Experiment 1 Contribution
Experiment 1 established the Cropchain project baseline by defining the business problem, stakeholder use cases, functional requirements, non-functional requirements, and requirement-phase effort estimates. It created the core planning artifact that guided all downstream work.

## 2. Experiment 2 Contribution
Experiment 2 extended the baseline into design and implementation planning. It translated requirements into a system architecture, defined major components and interfaces, and produced an implementation strategy covering frontend, backend, APIs, AI services, and non-functional architecture considerations.

## 3. Experiment 3 Contribution
Experiment 3 completed the Waterfall flow by introducing validation and delivery planning. It generated requirement-aligned testing outputs and documentation planning outputs, ensuring the project remained coherent from requirements through implementation, testing, and documentation.

## 4. Coherence Across Experiments
The three experiments are coherent because each phase reused the saved outputs from earlier phases rather than generating isolated content. Requirements informed design, design informed implementation, and both informed testing and documentation.

## 5. Strongest Artifacts
Experiment 1 was strongest for baseline clarity and requirements traceability.
Experiment 2 was strongest for technical depth and architecture planning.
Experiment 3 was strongest for project completion because it connected validation, documentation, and presentation-level analysis.

## 6. Limitations to Mention
Some effort estimates and technical choices remain conceptual because the outputs were generated through LLM-guided planning rather than full implementation.
The architecture and documentation plans may require refinement in a real production environment.
Testing coverage is structured and requirement-aligned, but actual execution evidence would still be needed in a real system.
